# SmartDorm 角色权限

## 角色定义

| 角色标识 | 中文名称 | 说明 |
| --- | --- | --- |
| `student` | 学生 | 宿舍居住者，主要使用报修、进度和通知功能 |
| `dorm_manager` | 宿管员 | 负责楼栋日常管理、报修审核、通知和安全巡检 |
| `repair_worker` | 维修人员 | 负责接收和处理维修工单 |
| `logistics_admin` | 后勤管理员 | 全局运维管理者，掌握楼栋、报修、能耗和工单数据 |
| `system_admin` | 系统管理员 | 负责系统用户、角色、权限和审计 |

## 权限模型

系统采用 RBAC 模型：

- `users`：用户账号。
- `roles`：角色定义。
- `permissions`：权限点。
- `user_roles`：用户角色关联。
- `role_permissions`：角色权限关联。

权限命名建议采用 `domain:action` 格式，例如：

- `dashboard:read`
- `repair:create`
- `repair:approve`
- `work_order:assign`
- `energy:read`
- `system:user_manage`

## 数据范围

- 学生只能访问自己的入住、报修、评价和通知。
- 宿管员默认只能访问负责楼栋的数据。
- 维修人员只能访问分派给自己的工单及相关报修信息。
- 后勤管理员拥有全局业务数据权限。
- 系统管理员拥有系统配置权限，但业务数据修改应按授权控制。

## 审计要求

以下操作必须写入 `audit_logs`：

- 登录失败、权限拒绝等安全事件。
- 用户、角色、权限变更。
- 入住、退宿、床位调整。
- 报修审核、派单、工单完成和关闭。
- 能耗告警处理。
- 通知公告发布、撤回。
