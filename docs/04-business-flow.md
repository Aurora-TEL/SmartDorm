# SmartDorm 业务流程

## 登录流程

1. 用户输入账号和密码。
2. 后端校验账号状态和密码。
3. 后端返回 token、用户信息、角色和权限。
4. 前端保存 token，拉取当前用户和菜单。
5. 前端按主角色跳转推荐首页。
6. 任一接口返回 401 时，前端清除 token 并跳转登录页。

## 学生报修闭环

1. 学生选择宿舍房间、报修类型和紧急程度，填写描述后提交报修单。
2. 系统生成 `repair_reports`，初始状态为 `submitted`。
3. 宿管员或后勤管理员审核：
   - 通过：状态改为 `approved`，可生成维修工单。
   - 驳回：状态改为 `rejected`，记录驳回原因。
4. 后勤管理员或宿管员派单，生成或更新 `repair_work_orders`。
5. 维修人员接单，状态改为 `in_progress`。
6. 维修人员完成处理，填写结果、费用说明和图片说明，状态改为 `completed`。
7. 学生确认并评价，报修单状态改为 `closed`。
8. 全流程关键节点写入通知和审计日志。

## 工单处理流程

1. 待派工单进入维修工单池。
2. 管理员选择维修人员、计划时间和优先级。
3. 维修人员在个人任务页查看工单。
4. 维修人员可更新处理进度。
5. 完成后由学生或管理员确认关闭。
6. 超时或异常工单进入看板预警。

## 入住流程

1. 管理员创建或导入学生档案。
2. 选择校区、楼栋、楼层、房间和床位。
3. 系统校验床位状态为 `available`。
4. 创建 `check_ins` 记录，并将床位状态改为 `occupied`。
5. 退宿时写入退宿时间，床位恢复为 `available` 或 `maintenance`。

## 能耗监测流程

1. 管理员维护表计信息，绑定楼栋或房间。
2. 系统定期或手工录入能耗读数。
3. 后端计算区间消耗量和趋势数据。
4. 当读数异常、突增或超过阈值时生成 `energy_alerts`。
5. 宿管员或后勤管理员处理告警，记录原因和处理结果。

## 安全巡检流程

1. 宿管员创建巡检记录，选择楼栋和巡检日期。
2. 根据模板生成巡检项。
3. 宿管员逐项填写正常、异常或需整改。
4. 异常项记录问题描述、整改建议和图片说明。
5. 管理员跟进整改状态，完成后关闭异常。

## 通知公告流程

1. 管理员编辑公告标题、内容、范围和有效期。
2. 发布后生成公告记录，并按范围生成站内通知。
3. 用户在通知中心查看未读消息。
4. 用户可标记单条或全部已读。

## 错误处理流程

- 读取接口失败：页面显示真实错误，并允许重试。
- 写操作失败：保持原状态，不显示成功提示。
- 401：清除 token，跳转登录页。
- 403：显示无权限提示。
- mock fallback：仅在 `VITE_ENABLE_MOCK_FALLBACK=true` 时允许读取演示数据。
