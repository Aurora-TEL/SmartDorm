# SmartDorm 需求说明

## 项目定位

SmartDorm 是面向高校宿舍管理场景的智慧报修与能耗运维平台。它以中后台数据看板为核心，把宿舍资产、学生入住、报修工单、能耗异常、安全巡检和通知公告整合到一个统一系统中，用于展示完整的软件工程设计与可运行原型。

## 目标用户

- 学生：便捷提交宿舍报修、查看处理进度、接收通知、评价维修结果。
- 宿管员：管理楼栋日常事务，审核报修，发布通知，执行安全巡检。
- 维修人员：接收维修工单，处理维修任务，填写处理结果。
- 后勤管理员：全局管理楼栋、房间、入住、报修、能耗和工单。
- 系统管理员：维护用户、角色、权限和审计日志。

## 核心诉求

1. 降低学生报修沟通成本，形成可追踪的闭环流程。
2. 提升宿管和后勤对工单、能耗、巡检的可视化管理能力。
3. 支持多角色权限隔离，符合中后台系统的基本安全要求。
4. 通过看板和图表展示运营数据，适合复试答辩演示。
5. 使用 Docker Compose 提供可复现运行环境。

## 功能范围

### 必须完成

- 认证登录、当前用户、角色权限控制。
- 首页运营看板：报修、工单、能耗、入住、安全巡检核心指标。
- 宿舍楼宇、楼层、房间、床位基础管理。
- 学生档案、入住、退宿和床位占用状态。
- 报修提交、审核、派单、处理、确认和评价。
- 维修工单列表、状态流转和处理记录。
- 能耗表计、读数、异常告警和趋势图。
- 安全巡检计划、巡检项、异常记录。
- 通知公告发布和通知中心。
- 数据分析、用户角色权限、审计日志。

### 暂不展开

- 微信小程序、短信网关、IoT 硬件真实接入。
- 复杂计费、支付、采购、库存和第三方维修商管理。
- 大规模多租户、跨学校商业化部署。

## 非功能需求

- 易演示：提供清晰角色入口和典型业务数据。
- 可维护：后端按 API、schema、service、model 分层。
- 可扩展：数据库表保留状态、审计、软删除或扩展字段设计空间。
- 可观测：关键写操作记录审计日志。
- 可部署：所有服务通过 Docker Compose 启动。

## 前端体验要求

- 风格：浅色科技风、左侧导航、顶部栏、指标卡片、ECharts 图表。
- 信息密度：中后台风格，优先清晰、稳定、可扫描。
- 错误处理：正式模式显示真实错误，不用 mock 伪装成功。
- 认证处理：401 清除 token 并跳转登录页。
- 角色跳转：登录后按角色进入推荐首页。

## 成功标准

- 答辩时能够用 3 到 5 分钟讲清系统背景、架构、数据模型和核心流程。
- 能够启动容器并完成至少一条完整报修流程演示。
- 看板能展示来自后端的核心运营指标。
- 文档、数据库和 API 能互相对应。
