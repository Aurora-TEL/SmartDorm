# SmartDorm 功能模块

## 模块总览

| 模块 | 说明 | 主要角色 |
| --- | --- | --- |
| 认证与个人中心 | 登录、登出、当前用户、角色入口 | 全部 |
| 首页看板 | 核心指标、趋势图、待办和告警 | 宿管员、后勤管理员、系统管理员 |
| 楼宇房间管理 | 校区、楼栋、楼层、房间、床位 | 宿管员、后勤管理员 |
| 学生入住管理 | 学生档案、入住、退宿、床位分配 | 宿管员、后勤管理员 |
| 报修管理 | 学生报修、审核、状态跟踪、评价 | 学生、宿管员、后勤管理员 |
| 维修工单 | 派单、接单、处理、完成记录 | 维修人员、后勤管理员 |
| 能耗监测 | 表计、读数、趋势、楼栋对比 | 宿管员、后勤管理员 |
| 能耗告警 | 异常识别、处理、关闭 | 宿管员、后勤管理员 |
| 安全巡检 | 巡检任务、巡检项、隐患记录 | 宿管员、后勤管理员 |
| 通知公告 | 公告发布、范围控制、置顶 | 宿管员、后勤管理员、系统管理员 |
| 通知中心 | 站内消息、已读未读 | 全部 |
| 数据分析 | 报修、能耗、入住、巡检统计 | 后勤管理员、系统管理员 |
| 系统管理 | 用户、角色、权限、审计日志 | 系统管理员 |

## 认证与个人中心

- 用户使用账号密码登录，后端返回访问令牌和当前角色信息。
- 前端根据角色跳转：学生 `/student`，宿管员 `/manager`，维修人员 `/worker`，后勤管理员和系统管理员 `/dashboard`。
- 当前用户接口返回用户基础信息、角色、权限和可访问菜单。

## 首页看板

- 指标卡：今日报修、待处理工单、当前入住率、今日能耗、异常告警、巡检完成率。
- 趋势图：近 7 日报修趋势、近 30 日能耗趋势。
- 排名：高频报修楼栋、能耗异常楼栋。
- 待办：待审核报修、待派工单、未处理告警、巡检异常。

## 楼宇房间管理

- 校区、楼栋、楼层、房间和床位分层管理。
- 房间记录容量、用途、状态和所属楼栋楼层。
- 床位记录占用状态，并和学生入住记录关联。

## 学生入住管理

- 学生档案包含学号、姓名、学院、专业、班级、联系方式。
- 入住记录关联学生、房间、床位、入住时间和退宿时间。
- 入住时校验床位可用，退宿时释放床位。

## 报修管理

- 学生选择房间、报修类型、紧急程度，填写描述并提交。
- 宿管员或后勤管理员审核报修，可驳回或派发维修。
- 学生可查看进度，并在完成后评价。

## 维修工单

- 报修审核通过后生成维修工单。
- 工单可分配给维修人员，维修人员接单、处理、填写结果。
- 工单状态覆盖待派单、已派单、处理中、已完成、已关闭。

## 能耗监测

- 维护水表、电表等能耗表计。
- 定期记录读数，按楼栋、房间、表计类型汇总趋势。
- 支持看板和分析页展示能耗曲线、同比环比和异常提示。

## 安全巡检

- 宿管员按楼栋创建巡检记录。
- 巡检项覆盖消防、门禁、用电、卫生和设施。
- 异常巡检项可关联整改描述和处理状态。

## 通知公告与通知中心

- 公告面向全部、指定楼栋、指定角色或指定学生发布。
- 站内通知用于系统事件提醒，例如工单派发、报修进度更新、告警产生。
- 通知中心支持未读计数、标记已读和批量已读。
